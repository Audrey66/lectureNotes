The Algebra* justification works for most mathematical manipulations

Will not work on claims involving ∧, ∨, →, ∀, ∃



Subst_<(m, n)  and Subst_>(m, n)


m: line number with an equivalence, like:
    LHS_M == RHS_M


n: any statement



< or > points at the value being REPLACED on line m


Subst_<(m, n): replace each occurrence of LHS_M with RHS_M
Subst_>(m, n): replace each occurrence of RHS_M with LHS_M

If you change the value of variable "name", write that claim
using:

    Old(name)

in the very next proof block. Only ONE "Old" value exists
at a time.

var x: Z = 2

Deduce (
    1 ( x == 2) by premise
)


x = x + 2       

deduce (
    1 (Old(x) == 2) by premise, 
    2 ( x == Old(x) + 2) by premise,
    3 ( x == 4) by Algebra*(1,2)
)


The Old(name) refers to value of the variable just before
the latest update.

Goal: state that variable's current value WITHOUT using Old
by the end of next proof block


y = y + 4

deduce(
    1 ( y == (Old(y) + 4)) by premise // this works
)

x = x + 3

Deduce(
    //what "Old" values are available?
    1 ( y == (Old(y) + 4)) // wasnt most recent change 
    2 ( x == Old(x) + 3) by premise //works, most recent
)

if (C) {

        //what premises?  
        1 ( C ) by Premise, //from if condition
        2 ( P ) by Premise, // was true before, nothing changed
    )

    statements

    Deduce(
        1 ( Q ) by Justification,
        2 ( A ) by Justification
    )

} else {
    Deduce(
        //what premises? 
        1 ( !(C) ) by Premise, //C is false since in else
        2 ( P ) by Premise, // was true before if, nothing changed
        
    )

    statements

    Deduce(
        1 ( Q ) by Justification,
        2 ( B ) by Justification
    )
}

    //what premises?
    1 ( Q ) by Premise, // true in both branches, had to follow one of the branches 
    2 ( A | B ) by Premise, // A true in if, B true in else
